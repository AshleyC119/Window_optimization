## 4. 双阶段 BO-LGBM 代理优化

### 4.1 方法论动机

物理引擎单次 NSGA-II 仅需 ~2 分钟。代理模型在此问题上的价值不是加速，而是**验证"代理+精炼"范式的精度闭包**——在多窗联合布设等扩展场景下，这将是从 4 维到 40+ 维的 scalability 跳板。

### 4.2 阶段一：LGBM 训练

| 项目 | 值 |
|---|---|
| 数据 | 5000 LHS + 300 真实 NSGA-II Pareto + 2000 边界过采样 = 7300 |
| 模型 | LightGBM, Optuna 50 轮 BO 超参搜索 |
| Feasible MAE | 0.52% |
| R² all / R² feasible | 0.995 / 0.908 |

### 4.3 阶段二：Warm-start 物理精炼

LGBM 代理跑 AGE-MOEA (200 代, μs 推理) → 产出 300 个候选。取 250 精英 + 50 随机个体混合注入物理 AGE-MOEA，仅跑 20 代。

**为什么不全用代理结果**：代理模型在非凸衍射边界有 ~1.3% 的预测偏差，演化算法会将其放大为 ~6-7% 的优化 gap。**为什么不全用物理引擎从头跑**：需要 60,000 次评估 (200 代 × 300 个体)。Warm-start 从精英群出发，20 代即可逼近真值，物理评估压缩至 6,000 次 (10×)。

```
代理 AGE-MOEA (纯 LGBM, 0 次物理) → [有偏候选集]
    ↓
物理 warm-start (20 代 × 300 = 6,000 次, ~45s)
    ↓
最终 Refined Knee: 1.66 m² @ 9.92%

真实 NSGA-II (60,000 次, ~120s):        1.64 m² @ 9.97%
双阶段 gap: **1%**
```

### 4.4 与端到端物理 NSGA-II 对比

| | 纯物理 NSGA-II | 双阶段 (LGBM+Warm-start) |
|---|---|---|
| 物理评估次数 | 60,000 | **6,000** (节省 90%) |
| 最终 Knee | 1.64 m² | **1.66 m²** (差 1%) |
| 技术栈 | Pareto dominance | BO-LGBM + AGE-MOEA |
| 可扩展性 | 4 变量 OK, 40+ 变量不可行 | 代理端零物理成本, 仅精炼端付费 |
